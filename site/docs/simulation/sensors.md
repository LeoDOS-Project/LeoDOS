# Sensors and Actuators

NOS3 provides hardware simulators that behave like real devices — they register on virtual buses (UART, SPI, I2C) and respond to the same register-level protocols as the physical hardware. Flight software interacts with them through the same hwlib API it uses in flight.

## Attitude Determination

- **Coarse sun sensors (CSS)** — measure the sun vector for coarse attitude estimation. Multiple sensors provide coverage across the spacecraft body.
- **Fine sun sensors (FSS)** — higher-resolution sun vector for precise pointing.
- **Inertial measurement unit (IMU)** — gyroscope (angular rate) and accelerometer data. Used for attitude propagation between star tracker updates.
- **Star tracker** — determines attitude by matching observed star patterns against a catalog. Provides the most accurate attitude measurement.
- **Magnetometer** — measures Earth's magnetic field vector. Used for coarse attitude determination and magnetorquer control.

## Navigation

- **GPS receiver (NovAtel OEM615)** — provides position and velocity from the [42 orbital propagation](orbital-mechanics). In the [SpaceCoMP workflows](/spacecomp/use-cases/overview), the GPS position is how the satellite determines whether it is over an area of interest.

## Power

- **Electrical power system (EPS)** — simulates battery state (voltage, temperature, charge), solar array output, and power modes. The battery model tracks charge and discharge cycles. Solar array output depends on the sun vector and eclipse status from 42.

## Attitude Control

- **Reaction wheels** (3 units) — store angular momentum for fine pointing control. The flight software commands wheel speeds; the simulator computes the resulting torques and feeds them back to 42's attitude dynamics.
- **Magnetorquers** — generate torques against Earth's magnetic field for momentum desaturation. The simulator uses the magnetic field vector from 42.
- **Thrusters** — for orbit and attitude maneuvers. The simulator applies thrust vectors to 42's orbital and attitude dynamics.

## Imaging

- **Arducam (OV5640)** — visible-light camera over I2C/SPI. Provides basic imaging capability.
- **Thermal camera** — custom NOS3 component for thermal IR Earth observation. Serves synthetic brightness temperature frames generated by [eosim](earth-observation) over a virtual SPI bus. Detects AOI entry using the satellite's position from 42 and loads the appropriate frame for each pass.

## What Is Not Simulated

- **Radiation effects** — single-event upsets, total ionizing dose, and latchups are not modeled. The [fault tolerance](/cfs/mission/fault-tolerance) mechanisms that handle these in flight are not exercised in simulation.
- **Detailed thermal modeling** — the EPS provides fixed temperature values rather than a dynamic thermal model of the spacecraft.
- **Real modulation/demodulation** — the [physical layer](/protocols/physical/overview) (BPSK, QPSK, etc.) is handled by real radio hardware in flight. In simulation, communication bypasses the RF chain entirely.
